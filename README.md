## How I approach web security

I treat web applications as systems with logic, not as a list of vulnerabilities.

My approach is usually:
- observe requests and responses
- compare normal vs abnormal behavior
- question what the application trusts
- look for inconsistencies in flow and validation

Many of the issues I work on are not about complex exploits, but about **broken assumptions**.

---

## Why web application security matters to me

Most real world attacks still start at the web layer.

Even strong technologies fail when:
- logic is flawed
- client input is trusted
- security controls are applied inconsistently

Understanding web security helps me think both offensively and defensively, and connects naturally with SOC and cloud security.
